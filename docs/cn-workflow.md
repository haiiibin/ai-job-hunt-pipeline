# 中文简历工作流（Chinese CV Workflow）

## 候选人基本信息（首次使用前填写）

生成任何中文简历前，先在此登记你的固定信息（与 `templates/cn_skeleton.html` 页眉占位符一致）：

- **姓名**：{{姓名}}（英文名可选）
- **学历**：{{最高学历院校 + 专业 + 学位 + 毕业年月}}；{{本科院校 + 专业 + 毕业年月}}
- **证书/成绩**：{{证书、语言成绩等，据实列出}}
- **联系方式**：{{手机号}} | {{邮箱}}
- **身份口径**：如已毕业，统一用"应届毕业生 / 职场新人"表述，禁止使用在读生表述

## 简历定制模型（动态生成，不维护多份固定版本）

**不维护按岗位类型分开的多份"固定模板"。** 所有内容按目标 JD **从 `resume-data/` 动态选取**（经历/项目按 tags 命中度筛选，该目录用户自建、已 gitignore，结构见仓库 CLAUDE.md），只保留少量按岗位类型切换的开关：

| 维度 | 规则 |
|---|---|
| 教育背景 | **首次填好后锁定**：`templates/cn_skeleton.html` 教育块填完即固定，所有岗位完全一致 |
| 字体/排版 | **锁定不变**：全宋体，CSS 已在骨架中写死（见下方"关键 CSS 结构"） |
| 经历/项目选取 | 读 `resume-data/_index.yaml`，priority=required 的经历全选；项目按 JD tags 重合度选 3-5 个 |
| 页眉 GitHub | 互联网/大厂/数据岗 → 加 `GitHub：github.com/{{GITHUB_ID}}`；其他不加 |
| 可选经历开关 | 与特定行业强绑定的经历（如券商实习）仅对口岗位包含；其他岗位删除该 job-entry 块 |
| 第三章节标题 | 互联网/数据/金融 → `项目经历`；咨询/管培 → `商赛经历`；数据工程 → `竞赛经历` |
| 技能区 | 按岗位侧重重排/裁剪 `<li>`，每行收成干净单行，避免"一行多一两个字"的孤尾 |
| bullet 描述 | **以资深 HR 的视角撰写**：从 `resume-data/` 的真实细节展开，每条**约 1.5 行**（约 55-72 中文字含加粗标签）；**不要写满两行或更长**，超过就精简，保留最有力的指标与动作；长度统一、不空泛；不出现破折号；标签不用括号注释 |

**中文简历来源文件**：`resume-data/_index.yaml`（索引入口）+ `resume-data/experience/*.md` 和 `resume-data/projects/*.md`（按 tags 按需读取中文 section）。可自备一份满意的历史简历 PDF 作为**格式基准**（字体/间距/排版参照它）。

**照片**：放置于仓库根 `photo.jpg`（用户自备，已 gitignore），模板以相对路径 `../../photo.jpg` 引用（相对 `output/[公司]_[职位]/` 目录）

## 执行流程

**第一步：判断投递市场**
- 国内岗位（中国大陆企业）→ 输出中文邮件
- 北美岗位 → 输出英文 Cover Letter（按英文 CL 规范执行）

**第二步：企业与岗位背调**（网络搜索）
- 主营业务、行业定位、近期动态（融资/新产品/战略转型）
- 企业文化与竞争定位
- 岗位在公司中的定位与核心职责

**第三步：JD 关键词拆解**
- 硬技能（工具/语言/模型/证书）
- 软技能（沟通/协作/抗压）
- 优先加分项

**第四步：内容匹配**
- 读 `resume-data/_index.yaml`，按 JD 类型选取最匹配的经历和项目文件（读中文 section）
- 判断小差异（单次内容决策，无需评分表）：
  - 页眉：互联网/数据岗添加 GitHub；其他岗位不加
  - 课程行：按岗位方向选择对应课程组合
  - 第三章节标题：互联网/数据→项目经历；咨询/管培→商赛经历；数据工程→竞赛经历
  - 可选经历：仅对口岗位包含（如券商实习只进金融/投行/资管岗），其他删除该 job-entry 块

**第五步：推荐结论**
1. 内容选取方案 + 理由（≤3条，精准对应JD关键词）
2. 临时调整建议（某条 bullet 需微调则具体指出）
3. 投递材料核心论点

**第六步：自我介绍**
150-200字，岗位定制版，含学历背景、核心技能、代表经历量化成果、应聘动机（结合背调结果，不得泛泛而谈）

**第七步：生成简历 + 投递材料**（见下方）

**第八步：满页验证（强制，量化检查）**
编译完成后必须执行：
1. **量化测量填充率**（不靠目测）：用 PyMuPDF 渲染第一页并读最后一行文字的 y 坐标：
   ```bash
   python -c "import fitz; p=fitz.open('<pdf路径>')[0]; m=max(w[3] for w in p.get_text('words')); print('fill%:', round(m/p.rect.height*100,1))"
   ```
   - **目标 fill% ≥ 93%**（A4 满页）。< 93% 视为留白过多，必须补内容。
2. fill% 不足时，按以下顺序补足（禁止用空白行或装饰元素填充）：
   - 把**置顶 1-2 段经历**（与 JD 最相关的）从 2 条扩为 3 条 bullet（从对应 `resume-data/experience/*.md` 中文 section 取真实内容）
   - 把偏短的 bullet 补到约 1.5 行（以 HR 视角展开真实细节，不虚构；**不要超过两行**）
   - 第三章节项目 bullet 也写到约 1.5 行
   - 仍不足再新增一个项目/竞赛条目
3. 重新编译，重测 fill%，直到 ≥ 93% 且仍为单页（pypdf 校验 pages==1）
4. 满页后保留 `.html` 与 `.pdf`

## 中文简历生成规则

**文件组织**：每个JD在 `output/[公司名]_[职位]/` 下创建文件夹，文件命名：
```
output/[公司]_[职位]/[你的姓名]_CV_[职位].html
output/[公司]_[职位]/[你的姓名]_CV_[职位].pdf
```

**HTML 模板**：使用 `templates/cn_skeleton.html`（唯一骨架，CSS 完全锁定）
- **生成方式**：Read `templates/cn_skeleton.html`，只填写注释占位符区域，**禁止修改 `<style>` 块**
- 所有字体、颜色、布局已预设在骨架中，生成时不得重写或调整任何 CSS
- 第三章节标题按岗位替换：互联网/数据→项目经历；咨询/管培→商赛经历；数据工程→竞赛经历
- 联系方式占位符已在骨架页眉中；互联网/数据岗需替换为含 GitHub 的格式

**内容保真规则（生成时严格遵守）**：

- **公司名与职位名使用锁定表**：首次使用时，把你每段经历的公司中文名（job-left）与职位中文名（job-center）登记成一张锁定表（放进 `resume-data/_index.yaml` 或单独文件），之后所有简历只用锁定版本，**禁止每次生成时临时翻译**。示例结构：

  | 实习经历 | 公司名（job-left） | 职位名（job-center） |
  |---|---|---|
  | 经历1（当前，置顶） | 公司中文名（English Name Inc.） | 职位中文名 |
  | 经历2 | 公司中文名（English Name） | 职位中文名 |
  | 可选经历 | 公司中文名 | 职位中文名（仅对口岗位包含，附日期区间） |

  | 竞赛/项目 | 名称（job-left） | 角色（job-center） |
  |---|---|---|
  | 竞赛1 | 竞赛全名（主办方+性质） | 项目负责人 |

  **所有竞赛/项目条目角色统一为"项目负责人"**（或你锁定的其他统一表述），不得逐份简历漂移。

- **填满整页**：编译后若底部有明显空白，按以下顺序补足：
  1. 将某条经历的 2 条 bullet 扩为 3 条（从对应 `resume-data/experience/*.md` 中文 section 选取真实内容）
  2. 在第三章节新增一个竞赛/项目条目
  3. 适当扩写已有 bullet（不虚构，仅展开已有描述）
  - 不得用空白行或装饰元素填充

**关键 CSS 结构（不可修改）**：
- 页面：`@page { size: A4; margin: 0.3cm 1.06cm 0.45cm 1.06cm; }`
- **字体（全宋体）**：`body { font-family: 'SimSun', 'NSimSun', '宋体', serif; font-size: 9.5pt; line-height: 1.38; color: #1a1a1a; }`。中文与英文/数字**统一走宋体**（不要 Times New Roman，更不要微软雅黑等无衬线）
- 姓名：`.header-name { letter-spacing: 2pt; }`。紧凑，禁止改宽
- 三列标题比例：`.job-heading { grid-template-columns: 2fr 1.25fr 1fr; }`。左列加宽以容纳"公司中文名（English Name Inc.）"这类长公司名（宋体英文较宽），右列因只放日期可收窄。**严禁用 position:absolute**，Chrome headless print 下绝对定位元素脱离文档流
- 页眉间距：`.header { margin-bottom: -14pt; }`。负边距让"教育背景"线贴近照片底部，**禁止改为正值**
- 照片：`.header-photo { width: 2.6cm; height: 3.3cm; overflow: hidden; justify-self: end; }`。**严禁用 position:absolute**，会破坏 grid 布局
- 右列样式：`.job-right { text-align: right; font-size: 8.5pt; }`。继承 bold 和黑色，**不得单独设置 font-weight 或 color**
- 右列内容格式：**只放日期，单行，不加城市/国家**。教育条目用 `毕业时间:YYYY年MM月`（半角冒号）；工作/实习/项目条目用纯日期区间（如 `2025年12月–至今`、`2024年10月`）
- 章节标题：`h2 { color: #1a1a1a; border-bottom: 1.2pt solid #1a1a1a; }`。必须黑色，**禁止红色或其他颜色**

**编译命令**（在仓库根执行）：
```bash
python compile_cv.py output/[公司]_[职位]/[你的姓名]_CV_[职位].html
```
compile_cv.py 自动识别 `.html` 后缀，使用 Chrome headless `--print-to-pdf`。

**单页验证**：
- Chrome 按 `@page` A4 设置打印，内容超出自动分页
- 编译成功后 PDF 约 350-450 KB（正常单页）
- 内容过多溢出第二页 → 减少 bullet 数量（3条改2条）或缩短 bullet 文字
- 内容太少底部空白 → 增加 bullet 内容或增加第三章节条目
- 每个实习/工作经历**最少 2 条 bullet**，硬性要求

**版本特定内容**（生成时严格遵守）：

*页眉联系方式*：
- 互联网/数据岗：`手机：{{手机号}} | 邮箱：{{邮箱}} | GitHub：github.com/{{GITHUB_ID}}`
- 其他岗位：`{{手机号}}|{{邮箱}}`（竖线两侧无空格）

*教育背景（首次填好后锁定，所有岗位一致）*：
直接用 `templates/cn_skeleton.html` 中已填好的教育块，**不随岗位变化**。填写要点：
- 每个学位一个 job-entry：job-left 放院校（可附一个客观加分标签，如世界排名）、job-center 放 `专业-学位层次`、job-right 放 `毕业时间:YYYY年MM月`
- 硕士 bullet：GPA、证书、语言成绩等亮点收成单行
- 本科 bullet 1：GPA、辅修；bullet 2：主修课程（可本硕合列，按岗位方向选课程组合）
- 课程作为本科的第 2 条 bullet（非 course-line）

*可选经历*：与特定行业强绑定的经历仅对口岗位包含，其他岗位删除该 job-entry 块

*第三章节标题*：
- 互联网/数据/金融岗：`项目经历`
- 咨询/管培岗：`商赛经历`（完整竞赛条目，每个 1 条 bullet）
- 数据工程岗：`竞赛经历`（精简版 bullet）

*技能区*：按岗位类型替换 `<ul class="skills-list">` 内全部 `<li>` 内容

## 国内投递邮件规范

**主题行**：优先遵循公司规定格式；无规定时参考：
`应聘[岗位名称]-{{姓名}}-{{院校+学位}}-[核心亮点标签]`

**正文结构（总字数300-400字）**：

① 开头（3-4句）：明确应聘岗位及信息来源；结合背调说明具体兴趣原因（禁止泛泛而谈）；一句话介绍自身背景

② 正文亮点（2-3段）：每段一个与JD高度匹配的核心能力；结合具体经历和量化数据；内容与简历互补，禁止直接复制简历原文

③ 结尾（2-3句）：明确再次表达意向；说明附件内容；礼貌请求回复，留联系方式

**格式规范**：
- 直接使用公司名称，不用"贵公司"
- 禁止空洞套话，所有能力描述需有具体事例支撑
- 署名：{{姓名}} | {{手机号}} | {{邮箱}}
